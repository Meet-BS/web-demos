Comprehensive testing suite for XML sitemap authentication scenarios

üéØ Overview

This testing suite demonstrates four different authentication scenarios for XML sitemaps:

  • Root auth, children open: Index requires auth, individual sitemaps are public
  • Root auth, children auth: Both index and individual sitemaps require auth
  • Root open, children open: All sitemaps are publicly accessible
  • Root open, children auth: Index is public, individual sitemaps require auth

Automated Testing Script

Copy and paste this script to test all scenarios at once:

for mode in root-only both none children-only; do echo "=== Testing $mode ===" echo "Root without auth: $(curl -o /dev/null -s -w '%{http_code}' https://web-demos-9iun.onrender.com/sitemap-scenarios/$mode/sitemap-index.xml)" echo "Root with auth: $(curl -o /dev/null -s -u admin:admin -w '%{http_code}' https://web-demos-9iun.onrender.com/sitemap-scenarios/$mode/sitemap-index.xml)" echo "Child without auth: $(curl -o /dev/null -s -w '%{http_code}' https://web-demos-9iun.onrender.com/sitemap-scenarios/$mode/pages.xml)" echo "Child with auth: $(curl -o /dev/null -s -u admin:admin -w '%{http_code}' https://web-demos-9iun.onrender.com/sitemap-scenarios/$mode/pages.xml)" echo done

Environment Variable Modes:

  • SITEMAP_AUTH_MODE=both - Both root and children require auth
  • SITEMAP_AUTH_MODE=root-only - Only root requires auth
  • SITEMAP_AUTH_MODE=children-only - Only children require auth
  • SITEMAP_AUTH_MODE=none - No authentication required (default)
