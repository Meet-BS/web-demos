This demonstrates a blocking UI overlay that appears when a specific cookie is not set. Perfect for first-time user experiences or feature gates.

How it works:
When the "existingUser" cookie is not set, a blocking overlay will appear. Clicking "Continue" sets the cookie and prevents the overlay from showing again.

Blocking UI Features:

  • Cookie-based state management
  • Full-screen blocking overlay
  • Persistent user preferences
  • First-time user experience
  • Easy cookie management
  • No server-side sessions needed
