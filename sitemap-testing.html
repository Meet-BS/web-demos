Comprehensive testing suite for XML sitemap authentication scenarios

üéØ Overview

This testing suite demonstrates four different authentication scenarios for XML sitemaps:

  • Root auth, children open: Index requires auth, individual sitemaps are public
  • Root auth, children auth: Both index and individual sitemaps require auth
  • Root open, children open: All sitemaps are publicly accessible
  • Root open, children auth: Index is public, individual sitemaps require auth

üîó Multi-Scenario Endpoints

All four authentication scenarios are available simultaneously without server restarts:

Root-Only Protection

Index requires auth, children are open

/sitemap-scenarios/root-only/sitemap-index.xml

Both Protected

Both index and children require auth

/sitemap-scenarios/both/sitemap-index.xml
/sitemap-scenarios/both/pages.xml

None Protected

All endpoints are publicly accessible

/sitemap-scenarios/none/sitemap-index.xml
/sitemap-scenarios/none/pages.xml

Children-Only Protection

Index is open, children require auth

/sitemap-scenarios/children-only/sitemap-index.xml
/sitemap-scenarios/children-only/pages.xml

Available Child Groups

Each scenario supports these child sitemap groups:

  • pages.xml - Main site pages (/, /blocking-ui, /slow, /iframe-demo.html)
  • auth-core.xml - Authentication demos (/basic-auth/, /form-auth, etc.)
  • auth-flows.xml - Authentication flow pages (/form-auth/login, etc.)

üîß Manual Testing Commands

Individual Scenario Testing

Test each scenario manually using curl commands:

# Root-Only Scenario (root auth, children open) curl -i /sitemap-scenarios/root-only/sitemap-index.xml # Should get 401 curl -i -u admin:admin /sitemap-scenarios/root-only/sitemap-index.xml # Should get 200 curl -i /sitemap-scenarios/root-only/pages.xml # Should get 200 # Both Protected Scenario curl -i /sitemap-scenarios/both/sitemap-index.xml # Should get 401 curl -i -u admin:admin /sitemap-scenarios/both/sitemap-index.xml # Should get 200 curl -i /sitemap-scenarios/both/pages.xml # Should get 401 curl -i -u admin:admin /sitemap-scenarios/both/pages.xml # Should get 200 # None Protected Scenario curl -i /sitemap-scenarios/none/sitemap-index.xml # Should get 200 curl -i /sitemap-scenarios/none/pages.xml # Should get 200 # Children-Only Scenario curl -i /sitemap-scenarios/children-only/sitemap-index.xml # Should get 200 curl -i /sitemap-scenarios/children-only/pages.xml # Should get 401 curl -i -u admin:admin /sitemap-scenarios/children-only/pages.xml # Should get 200

Automated Testing Script

Copy and paste this script to test all scenarios at once:

for mode in root-only both none children-only; do echo "=== Testing $mode ===" echo "Root without auth: $(curl -o /dev/null -s -w '%{http_code}' /sitemap-scenarios/$mode/sitemap-index.xml)" echo "Root with auth: $(curl -o /dev/null -s -u admin:admin -w '%{http_code}' /sitemap-scenarios/$mode/sitemap-index.xml)" echo "Child without auth: $(curl -o /dev/null -s -w '%{http_code}' /sitemap-scenarios/$mode/pages.xml)" echo "Child with auth: $(curl -o /dev/null -s -u admin:admin -w '%{http_code}' /sitemap-scenarios/$mode/pages.xml)" echo done

üìÅ Static Sitemap Endpoints

Additional static sitemap files are also available (auth controlled by SITEMAP_AUTH_MODE environment variable):

Auth Core

/sitemaps/sitemap-auth-core.xml

Auth Flows

/sitemaps/sitemap-auth-flows.xml

Environment Variable Modes:

  • SITEMAP_AUTH_MODE=both - Both root and children require auth
  • SITEMAP_AUTH_MODE=root-only - Only root requires auth
  • SITEMAP_AUTH_MODE=children-only - Only children require auth
  • SITEMAP_AUTH_MODE=none - No authentication required (default)
